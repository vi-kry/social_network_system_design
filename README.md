# Social Network System Design

## Функциональные требования:
    - приложение поддерживает только русский язык
    - добавление собственных постов о путешествии вручную 
    - просмотр списка постов других путешественников
    - добавление лайков под постом
    - просмотр списка лайков под постом
    - добавление комментария под постом
    - просмотр списка комментариев под постом
    - подписка на других путешественников (write)
    - просмотр списка подписчиков
    - можно удалять посты
    - можно отписываться от путешественников
    - можно удалять собственные лайки
    - можно удалять собственные комментарии
    - фото к посту прикладывать необязательно для его публикации
    - можно только лайкать посты, дизлайкать нельзя (аналогично YouTube) 
    - пагинация по постам - 10 штук за раз
    - пагинация по лайкам - 20 штук за раз
    - пагинация по комментариям - 10 штук за раз
    - пагинация по подписчикам - 20 штук за раз

## Нефункциональные требования:
    - DAU 10 000 000
    - Аудитория стран СНГ
    - Данные будем хранить всегда
    - Активности пользователей
        - в среднем каждый пользователь добавляет 1 пост в месяц
        - в среднем каждый пользователь просматривает 40 постов в день
        - в среднем каждый пользователь оставляет 5 комментариев в день
        - в среднем каждый пользователь просматривает 30 комментариев в день
        - в среднем каждый пользователь делает 20 лайков в день
        - в среднем каждый пользователь просматривает 15 лайков в день
        - в среднем каждый пользователь подписывается на 2 людей в неделю
        - в среднем каждый пользователь просматривает список подписчиков 1 раз в день
        - в среднем каждый пользователь отписывается от людей 4 раза за полгода
        - в среднем каждый пользователь удаляет 1 пост раз в 6 месяцев
        - в среднем каждый пользователь удаляет 2 комментарий раз в месяц
        - в среднем каждый пользователь удаляет 1 лайк раз в 6 месяцев
    - Лимиты
        - нет лимитов на посты
        - пользователь не может подписываться на более чем 50 человек в сутки (накрутка подписчиков)
        - пользователь не может ставить более 200 лайков в сутки (накрутка лайков)
        - пользователь не может ставить более 1000 комментариев в сутки (фейковая активность в комментариях)
    - Сезонность
        - Активность пользователей повышается с мая по август (летний период), коэффициент на х3 на все пункты
        - Активность пользователей повышается с декабря по январь (зимний период), коэффициент х2 на все пункты
    - Тайминги
        - добавление поста не более 1 секунды
        - получение списка постов не более 2 секунд
        - добавление лайка не более 1 секунды
        - получение списка лайков под постом не более 2 секунд
        - добавление комментария под постом не более 1 секунды
        - получение списка комментариев под постом не более 2 секунд
        - подписка на других путешественников не более 1 секунды
        - получение списка подписчиков не более 2 секунд
        - удаление поста не более 1 секунды
        - отписка от путешественников не более 1 секунды
        - удаление собственного лайка не более 1 секунды
        - удаление собственного комментария не более 1 секунды


## Оценка нагрузки
> по поводу удаления постов/комментариев/лайков/отписок - не особо знаю куда и как считать

### Посты
    - RPS(write/create) = 10 000 000 * 1 / (30 * 24 * 60 * 60) ≈ 4
    - RPS(write/delete) = 10 000 000 * 1 / (180 * 24 * 60 * 60) ≈ 1
    - RPS(read) = 10 000 000 * 40 / (24 * 60 * 60) ≈ 4700
### Комментарии
    - RPS(write/create) = 10 000 000 * 5 / (24 * 60 * 60) ≈ 600
    - RPS(write/delete) = 10 000 000 * 2 / (30 * 24 * 60 * 60) ≈ 8
    - RPS(read) = 10 000 000 * 30 / (24 * 60 * 60) ≈ 3500
### Лайки
    - RPS(write/create) = 10 000 000 * 20 / (24 * 60 * 60) ≈ 2400
    - RPS(write/delete) = 10 000 000 * 1 / (180 * 24 * 60 * 60) ≈ 1
    - RPS(read) = 10 000 000 * 15 / (24 * 60 * 60) ≈ 1800
### Подписчики
    - RPS(write/create) = 10 000 000 * 2 / (7 * 24 * 60 * 60) ≈ 40
    - RPS(write/delete) = 10 000 000 * 4 / (180 * 24 * 60 * 60) ≈ 3
    - RPS(read) = 10 000 000 * 1 / (24 * 60 * 60) ≈ 120


## Таблицы в БД
    Post (
        post_id     uuid ≈ 16 bytes
        description text ≈ 50 bytes
        location_id uuid ≈ 16 bytes
        comments    uuid[] ≈ 344 byte (24 bytes + (16 byte * 20), где 20 - кол-во комментов
        created_at  timestamp ≈ 8 bytes
        ) ≈ 434 bytes

    Post_photo (
        post_photo_id uuid ≈ 16 bytes
        post_id       uuid ≈ 16 bytes
        order         integer ≈ 4 bytes
        url           text ≈ 16 bytes
        width         integer ≈ 4 bytes
        height        integer ≈ 4 bytes
        created_at    timestamp ≈ 8 bytes
    ) ≈ 68 bytes

    Location (
        location_id uuid ≈ 16 bytes
        locality    text ≈ 32 bytes
        latitude    double precision ≈ 8 bytes
        longitude   double precision ≈ 8 bytes
    ) ≈ 64 bytes
    
    User (
        user_id      uuid ≈ 16 bytes
        posts        uuid[] ≈ 344 byte (24 bytes + (16 bytes * 20), где 20 - кол-во постов
        subscribers  uuid[] ≈ 824 byte (24 bytes + (16 bytes* 50), где 50 - кол-во подписчиков
        subcriptions uuid[]≈ 824 byte (24 bytes + (16 bytes * 50), где 50 - кол-во подписок
        created_at   timestamp ≈ 8 bytes
    ) ≈ 2016 bytes
    
    Comment (
        comment_id uuid ≈ 16 bytes
        post_id    uuid ≈ 16 bytes
        content    text ≈ 50 bytes
        created_at timestamp ≈ 8 bytes
    ) ≈ 90 bytes
    
    Post_like (
        post_like_id uuid ≈ 16 bytes
        post_id      uuid ≈ 16 bytes
        user_id      uuid ≈ 16 bytes
        created_at   timestamp ≈ 8 bytes
    ) ≈ 56 bytes

## Трафик

### Посты
    - Traffic(write) = 4 RPS * (434 B + 68 B + 64 B) = 2830 B/s ≈ 2.9 kB/s
    - Traffic(read) = 4700 RPS * (434 B + 68 B + 64 B) * 10 шт. ≈ 26 602 kB/s
### Комментарии
    - Traffic(write) = 608 RPS * 90 B ≈ 55 kB/s 
    - Traffic(read) = 3500 RPS * 90 B * 10 шт. ≈ 3150 kB/s
### Лайки
    - Traffic(write) = 2401 RPS * 56 B ≈ 135 kB/s
    - Traffic(read) = 1800 RPS * 56 B * 20 шт. ≈ 2016 kB/s
### Подписчики
    - Traffic(write) = 43 RPS * 2016 B ≈ 87 kB/s
    - Traffic(read) = 120 RPS * 2016 B * 20 шт. ≈ 5000 kB/s
